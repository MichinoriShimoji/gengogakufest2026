1. 圧縮ベースの位置規則性分析

gzip 圧縮サイズでターゲット出現位置の規則性を評価。シャッフル帰無分布 N=500。Bonferroni 補正（k=3: 3形式 × 1シャッフル種）。

1.0 方法の仕組み

各発話の語列を、ターゲットなら TGT_、非ターゲットならその word_pos（先頭4文字）に符号化する。

例: 発話 V mmja N CP V
元データV___TGT_N___CP__V___
shuf 例1V___N___TGT_CP__V___
shuf 例2TGT_N___V___CP__V___

語を入れ替えるのではなく、TGT_ マーカーの位置をランダムに選び直す。非ターゲット語の POS 列は固定のまま。gzip はバイト列の繰り返しパターンを検出するため、TGT_ が特定の POS 文脈に偏って出現するほど圧縮が効き、サイズが小さくなる。シャッフルでその噛み合いが壊れると圧縮サイズが増大する。

uniform シャッフルは発話内の全位置から等確率で選ぶ。

1.2 unu が n.s. である理由

unu は直後に N が 45% と強い文脈選好を示すが（§2 参照）、圧縮検定では非有意である。これは「POS 文脈の偏り」と「文内位置の規則性」が異なる性質だからである。

unu の per-token 効果量は mmja の 53% あり、効果自体はゼロではない。しかし n=40 では絶対的な圧縮差 |Δ|=8.5 bytes がシャッフルノイズ σ=7.1 に埋もれる。仮に unu が n=170 あれば projected z ≈ −2.5 となり、補正前有意水準に近づく。

「文脈選好」と「位置規則性」は別。unu の「直後 N=45%」は POS 文脈の偏りであり、文の前半でも後半でも N の前に unu が出るなら同じ文脈になる。圧縮検定が測るのは「POS 列のどの位置に TGT_ が嵌まるか」のパターンであり、文脈の偏りとは独立。なお unu はサプライザル（§4）では有意（d=+0.46, p=.0005）であり、「どこに出るか」は不明瞭でも「出た後に情報量の高い語が来る」ことは確認される。
